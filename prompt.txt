你正在维护 RSR 播种监测系统（Vue3 + Vite + TS + FastAPI + PostgreSQL）。

目标：
- 保持项目可持续上线：配置收敛、安全增强、稳定运行、文档一致。
- 所有实现以“可部署、可联调、可维护”为优先。

## 强约束

1. 生产域名/IP 只允许出现在配置文件（`.env`、`frontend/.env` 及其 `.example`）。
2. 前端默认使用相对路径：
   - HTTP：`/api/v1`
   - WebSocket：按 `window.location` 推导 `ws/wss`，仅在 `VITE_WS_BASE` 配置时覆盖。
3. 鉴权统一为“管理会话 + ingest Bearer”：
   - 浏览器管理能力先通过 `POST /api/v1/auth/session` 提交 `ADMIN_TOKEN` 建立 HttpOnly Cookie
   - `INGEST_TOKEN`：仅 ingest，继续使用 Bearer
4. 数据分区使用固定表 + `run_id`：`runs`、`telemetry_samples`、`gps_points`。
5. 数据保留 30 天，后台任务每日清理。
6. 历史/备份目录保留但隔离，不参与构建发布。

## 接口约束（当前版本）

- `POST /api/v1/runs/start`
- `POST /api/v1/runs/{run_id}/stop`
- `GET /api/v1/runs`
- `GET /api/v1/runs/{run_id}`
- `DELETE /api/v1/runs/{run_id}`
- `GET /api/v1/runs/{run_id}/gps`
- `GET /api/v1/runs/{run_id}/telemetry`
- `GET /api/v1/runs/{run_id}/report.pdf`
- `POST /api/v1/ingest`
- `POST /api/v1/auth/session`
- `GET /api/v1/auth/session`
- `DELETE /api/v1/auth/session`
- `GET /ws/live?run_id=...`

## 前端约束

1. 监控页保持核心布局稳定，不做破坏式重排。
2. 必须有可感知的状态反馈：加载态、禁用态、失败提示、空态、断连态。
3. 历史页支持：任务列表、轨迹回放、报告查看、PDF/Excel 导出。
4. 地图依旧通过 `index.html` 注入 BMapGL，代码仅使用 `window.BMapGL`。

## 后端约束

1. `ingest` 仅做校验和入队，异步 worker 批写。
2. 队列高水位触发聚合背压（telemetry 取最后值，gps 每秒最后一条）。
3. WebSocket 服务端需做鉴权与失活连接清理。
4. 启动流程以 Alembic 为主，必要时 bootstrap 兜底，保证服务可启动。
