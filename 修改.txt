### 2) `backend/app/api/routes.py`

**要改什么**

把 `start_run()` 改成“**插入优先，冲突回查**”，不要再先查 active run 再插入。现在的实现是：

* 先 `_get_active_run_id(session)`
* 如果有，直接返回现有 run
* 没有才 `session.add(run)`
* `IntegrityError` 后再回查

这个逻辑比最早那版好，但还是保留了“先查再建”的竞态窗口。([GitHub][1])

**怎么改**

把 `start_run()` 改成：

1. 直接创建 `run`
2. 直接 `commit`
3. 如果成功，返回新 run
4. 如果 `IntegrityError`，回查当前 active run 并返回

也就是把这段：

```python
existing_run_id = await _get_active_run_id(session)
if existing_run_id:
    ...
session.add(run)
try:
    await session.commit()
except IntegrityError:
    ...
```

改成：

```python
now = datetime.now(timezone.utc)
run = Run(...)
session.add(run)

try:
    await session.commit()
    await session.refresh(run)
    return ...
except IntegrityError:
    await session.rollback()
    existing_run_id = await _get_active_run_id(session)
    ...
```

**顺手一起改**

把 `get_telemetry()` 默认 bucket 从硬编码 `"5s"` 改成读取 `settings.telemetry_history_bucket_seconds`。你配置里已经有这个字段，但路由默认值仍然写死成 `"5s"`。([GitHub][1])

建议改成：


**验收**

数据库唯一索引 + 新 `start_run()` 一起上线后，接口并发下不会再出现双 active run；改配置里的 `TELEMETRY_HISTORY_BUCKET_SECONDS`，历史遥测默认粒度会跟着变。([GitHub][1])

### 4) `frontend/nginx.conf`（新建）

**要改什么**

```nginx
    try_files $uri $uri/ /index.html;

    proxy_set_header Host $host;

}
```

### 8) `frontend/src/api/http.ts`

**要改什么**

现在这个文件已经很简洁：`axios.create({ withCredentials: true })`，并暴露了 `authApi.status/login/logout`。这条主线是对的。([GitHub][5])

但你可以再补两层小 patch：

第二层，给“会话已失效”提供全局回调，而不是在每个页面自己判断 `isAuthError()`。

**怎么改**

加一个响应拦截器：

```ts
http.interceptors.response.use(
  (resp) => resp,
  (error) => {
    const status = error?.response?.status
    if (status === 401 || status === 403) {
      window.dispatchEvent(new CustomEvent('rsr-auth-expired'))
    }
    return Promise.reject(error)
  }
)
```

然后登录页或全局 layout 里监听这个事件，弹登录框即可。

这样 `HistoryView.vue` 和 `useMonitorRealtime.ts` 里那种“各自判断 auth error 再 toast”的重复逻辑就可以慢慢收掉。([GitHub][4])
